# 운명 종합 분석 서비스 기획서

## 1. 프로젝트 개요

### 1-1. 서비스명
**운명 API (Unmyeong API)** — 샤머니즘 총집합 운세 서비스

### 1-2. 한줄 소개
생년월일시 + 혈액형만 입력하면 사주, 별자리, MBTI 예측, 혈액형 성격, 띠까지 한번에 분석하고 AI가 종합 해석해주는 REST API

### 1-3. 배경 및 목적
- MZ세대는 MBTI, 별자리, 사주, 혈액형 등 성격/운세 콘텐츠를 가볍게 소비
- 하지만 각각 따로 찾아봐야 하는 불편함 → **원스톱 종합 분석** 수요
- 개별 결과가 아닌, AI가 **모든 결과를 종합**해서 하나의 스토리로 풀어줌
- API 형태로 제공 → 앱/웹/챗봇 등 어디든 연동 가능

### 1-4. 타겟 사용자
- 운세/성격 분석에 관심 있는 20~30대
- 운세 기능을 자체 서비스에 연동하려는 개발자
- 친구/연인끼리 결과 공유하며 즐기고 싶은 사용자

### 1-5. 유저 입력 (최소화)
| 필드 | 필수 | 비고 |
|------|------|------|
| 생년월일 | 필수 | 양력 기준 |
| 태어난 시간 | 필수 | 모르면 시주 추정 퀴즈로 대체 |
| 혈액형 | 선택 | 모르면 해당 모듈 스킵 |

## 2. 분석 모듈

### 모듈 1: 사주팔자 (Four Pillars)
| 항목 | 설명 |
|------|------|
| 입력 | 생년월일시 (양력) |
| 출력 | 년주/월주/일주/시주, 천간·지지, 띠 |
| 로직 | 절기 기반 간지 계산 |

### 모듈 2: 오행 분석 (Five Elements)
| 항목 | 설명 |
|------|------|
| 입력 | 사주팔자 결과 |
| 출력 | 목/화/토/금/수 분포, 본인 오행, 강약 분석 |

### 모듈 3: 별자리 (Zodiac)
| 항목 | 설명 |
|------|------|
| 입력 | 생월일 |
| 출력 | 별자리, 원소(불/물/흙/바람), 수호성, 성격 키워드 |
| 로직 | 양력 날짜 기반 12별자리 매핑 |

### 모듈 4: 혈액형 성격 (Blood Type)
| 항목 | 설명 |
|------|------|
| 입력 | 혈액형 (A/B/O/AB) |
| 출력 | 성격 특징, 장점, 단점, 연애 스타일 |
| 로직 | 혈액형별 성격 상수 데이터 |

### 모듈 5: MBTI 예측 (MBTI Prediction)
| 항목 | 설명 |
|------|------|
| 입력 | 사주 + 별자리 + 혈액형 종합 결과 |
| 출력 | 예상 MBTI 타입, 각 축별 경향 점수 |
| 로직 | 오행·별자리 원소·혈액형 성격의 상관 매핑 |
| 참고 | 과학적 근거 X, 엔터테인먼트 목적 명시 |

### 모듈 6: 띠 운세 (Chinese Zodiac)
| 항목 | 설명 |
|------|------|
| 입력 | 생년 |
| 출력 | 띠 동물, 성격, 올해 운세, 궁합 띠 |
| 로직 | 12간지 동물 매핑 + 상수 데이터 |

### 모듈 7: 시주 추정 퀴즈 (Birth Hour Estimator)
| 항목 | 설명 |
|------|------|
| 입력 | 간단한 질문 4개에 대한 답변 |
| 출력 | 추정 태어난 시간대 (상위 2~3개 후보) |
| 로직 | 전통 명리학 기반 성향-시주 매핑 + AI 보정 |

**퀴즈 질문:**
| # | 질문 | 선택지 |
|---|------|--------|
| 1 | 형제/자매 중 몇째인가요? | 외동 / 첫째 / 중간 / 막내 |
| 2 | 아침형인가요, 저녁형인가요? | 아침형 / 저녁형 / 둘 다 아님 |
| 3 | 첫인상이 어떻다는 말을 자주 듣나요? | 차가움 / 따뜻함 / 활발함 / 조용함 |
| 4 | 의사결정 스타일은? | 즉흥적 / 신중함 / 상황에 따라 다름 |

### 모듈 8: AI 종합 해석 (AI Synthesis)
| 항목 | 설명 |
|------|------|
| 입력 | 모든 모듈의 분석 결과 |
| 출력 | 종합 성격, 올해 운세, 연애운, 직업운, 조언 |
| AI 모델 | Claude API |
| 톤앤매너 | 친근하고 재밌는 말투, 긍정적 방향 제시 |

## 3. API 설계

### 3-1. 엔드포인트

| Method | Path | 설명 | 인증 |
|--------|------|------|------|
| `GET` | `/health` | 서버 상태 확인 | 불필요 |
| `POST` | `/api/analyze` | 전체 모듈 분석 (AI 해석 제외) | 불필요 |
| `POST` | `/api/analyze/full` | 전체 분석 + AI 종합 해석 | API Key |
| `POST` | `/api/saju` | 사주팔자만 분석 | 불필요 |
| `POST` | `/api/zodiac` | 별자리만 분석 | 불필요 |
| `POST` | `/api/blood-type` | 혈액형 성격만 분석 | 불필요 |
| `POST` | `/api/estimate-hour` | 시주 추정 퀴즈 | 불필요 |
| `POST` | `/api/compatibility` | 두 사람 궁합 분석 | API Key |

## 4. 기술 스택

| 구분 | 기술 | 선정 이유 |
|------|------|-----------|
| 언어 | Python 3.11+ | 역학 관련 라이브러리 풍부 |
| 프레임워크 | FastAPI | Swagger 자동 문서화, 비동기, 타입 검증 |
| AI | Claude API (Anthropic) | 한국어 우수, 긴 컨텍스트로 종합 해석에 적합 |
| 음력 변환 | korean-lunar-calendar | 한국 음력/양력 변환 |
| 데이터 검증 | Pydantic v2 | FastAPI 기본 통합 |
| 서버 | uvicorn | ASGI 고성능 |

## 9. 리스크 & 고려사항

| 리스크 | 대응 |
|--------|------|
| 사주 계산 정확도 | 절기 기준 적용, 만세력 교차 검증 |
| Claude API 비용 | AI 해석 엔드포인트 분리, 개별 분석은 무료 |

| 혈액형 성격론 논란 | 과학적 근거 없음 명시, 재미 요소로만 제공 |
